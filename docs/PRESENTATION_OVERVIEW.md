# Egg Guardian - Project Overview

## Executive Summary

Egg Guardian is a production-grade IoT system for real-time temperature monitoring of commercial egg incubators. The system delivers sub-2-second alert latency from sensor to mobile device, enabling operators to respond immediately to temperature deviations that could harm embryo development.

## Key Features

### Real-time Monitoring
- Live temperature readings via WebSocket, updated every 1-2 seconds.
- 24-hour telemetry history with timestamp indexing.
- Automatic device list refresh every 5 seconds.

### Mobile Application (Flutter)
- Cross-platform: Android, Web, and iOS capable.
- JWT Authentication with secure session persistence.
- Live charts with safe-zone dynamic threshold lines.
- Alert banners for out-of-range readings.
- Firebase Cloud Messaging push notifications.

### Admin Dashboard (Web)
- Device registration and management.
- Per-device, dynamic alert rule configuration (min/max thresholds).
- Real-time triggered alerts dashboard, auto-refreshing.
- Full user management with role-based access.
- Last-admin protection to prevent self-lockouts.

### Backend API (FastAPI)
- 25+ REST endpoints with autogenerated Swagger documentation.
- Asynchronous WebSocket connections for live data streaming.
- Intelligent alert engine: evaluates each telemetry reading against stored rules.
- Dispatches SMTP email alerts and FCM push notifications on breach.

### IoT Firmware (ESP32)
- Reads DS18B20 1-Wire temperature sensor.
- Publishes telemetry to MQTT topic: `egg/{device_id}/telemetry`.
- Local buffer of 20 readings for network resilience.
- Automatic WiFi and MQTT reconnection logic.

## Security Features

- JWT token-based authentication with refresh tokens.
- Role-based access control (Standard User / Admin).
- Secure password hashing using Bcrypt.
- CORS protection with environment-configured origins.
- MQTT topic-based device access control.
- Secrets stored in `.env` (excluded from version control).
