01 / Background & Problem

The Agricultural & Security Gaps

  • Thermal Sensitivity: Embryos require tight boundaries ($37.0^\circ ext{C}$ to $39.0^\circ ext{C}$). Hypo/hyperthermia spikes cause complete embryonic mortality.
  • Outage Caching Gaps: Existing apps lack edge buffering. Agricultural Wi-Fi dropouts result in complete data gaps.
  • Firmware Vulnerabilities: IoT edge nodes run unauthenticated, outdated firmware without lifecycle warnings.
A single environmental crash or data spoofing attack on unencrypted channels (Blynk/analog setups) can cause total batch failure and extreme commercial losses.

03 / Engineering Design

System Architecture & Telemetry Flow

  • Edge Tier: DS18B20 sensor transmits via 1-Wire to ESP32 node.
  • Transport Tier: Telemetry published via MQTTS (TLS 1.3) on port 8883.
  • Cloud Tier: FastAPI backend processes telemetry, updates PostgreSQL, and broadcasts to active sockets.
  • HMI/Alerting Tier: Flutter UI refreshes in real time, with Firebase Cloud Messaging (FCM) pushing alerts.
04 / User Interfaces

Operator HMI Platforms

  • Authentication Cockpit: Employs JWT-driven session layers, storing secrets locally.
  • Temperature Detail View: Real-time interactive radial gauges changing color to reflect thermal status.
  • Admin Control Console: Web dashboard enabling threshold limit modification ($T_{ ext{min}}, T_{ ext{max}}$) and live telemetry log audits.

05 / Empirical Findings

System Metrics & Performance

  • Sensor Accuracy:DS18B20 calibration yielded a **Mean Absolute Error (MAE) of $0.24^\circ ext{C}$** (acceptable threshold: $\pm 0.5^\circ ext{C}$).
  • Edge Data Recovery: Ring buffer achieved **100% data recovery** during Wi-Fi drops under 100 seconds.
  • Data Processing Speed: WebSockets UI refresh occurred in **$142 ext{ms}$**; notification latency averaged $1.14 ext{s}$ over local Wi-Fi.

06 / System Validation

Security & Operator Usability
